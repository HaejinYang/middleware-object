# 배경

C++로 코테만 하다보니, 알던 것도 다 까먹고 재미가 없다. 
그래서, 머리 아프게 하는 자료구조나 알고리즘이 적용된 시스템을 분석하고, 수정도 해보려고 한다.
저런 것들을 통틀어서 뭐라고 해야할지 애매한데, 시작은 미들웨어 오브젝트라 프로젝트 이름이 미들웨어 오브젝트임.

# 내용

## 미들웨어 오브젝트

미들웨어(Middleware) 패턴을 쉽게 사용할 수 있는 클래스를 만든다.

## 미들웨어 패턴이란? 

미들웨어 패턴은 요청을 처리하는 일련의 파이프라인을 구성하는 방식이다. 
예를 들어, 웹 프로그래밍에서는 요청을 받아 인증된 사용자 여부를 검사한 뒤, 적절한 컨트롤러로 전달하는 과정이 이에 해당한다.


## 관련 구현

TupleUnpacker: 객체와 객체에서 호출하려는 메소드를 다른 객체에 담아서 나중에 호출할 수 있도록 하는 객체

CompensatoryTask: 수행할 Task과 Task가 실패했을 때 호출할 Task를 같이 저장하는 구조를 표현한 객체
